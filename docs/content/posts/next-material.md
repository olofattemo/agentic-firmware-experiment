## Agent Memory Mechanisms vs. Systems Engineering Equivalents

| Agent Mechanism | What It Does | SE Equivalent | What the Formal Approach Adds |
|---|---|---|---|
| `AGENTS.md` / agent rules | Project conventions and agent constraints | Coding standards, development guidelines | Separated by concern and versioned — not a flat file mixing style with safety |
| Skills / tool definitions | Domain-specific patterns and correct API usage | Design patterns, reference implementations | Traceable to requirements; explicit rationale, not just examples |
| Agent memory / session history | Incidental context accumulated during work | Structured persisted requirements | Memory drifts and compacts. Requirements documents do not |
